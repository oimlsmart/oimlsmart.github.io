## Five sections

### [Developer Guides](/docs/guides/getting-started.html)

The 13-guide narrative walkthrough — start here if you're new to SMART
authoring. Read sequentially or jump to any topic. Begins with creating
your first SMART Standard and ends with the semantic ontology layer.

### [Architecture](/docs/arch/overview.html)

The deep-dive into how the system is built — the six-layer entity model,
design principles, computation engine, expression language, tables &
lookups. For engineers who want to understand the engine internals.

### [Workflow](/docs/workflow/overview.html)

The end-to-end certification process — from manufacturer application to
certificate issuance. For OIML-CS participants, Issuing Authorities,
Test Laboratories, and anyone integrating with the certification
pipeline.

### [Formal Specifications](/docs/specifications/system-architecture.html)

The 11-document SMART_REQS set — the normative specifications that
define the platform itself. These are the equivalent of OIML B/D/R
documents but for the SMART platform. Authoritative, detailed, typed.

### [Reference](/docs/ref/yaml-schema.html)

Quick-reference material: YAML schemas, TypeScript type definitions, OCL
syntax, the URN specification. Lookup-on-demand material for working
developers.

## Also useful

## Standards already modelled

Three Recommendations are fully modelled: [R 60](/recommendations/r60.html)
(load cells), [R 129](/recommendations/r129.html) (mass road vehicles),
[R 144](/recommendations/r144.html) (gas meters). See the
[Adding a Standard](/docs/workflow/adding-a-standard.html) workflow guide
to model a new one.
